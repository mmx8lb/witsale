# Witsale 慧售 - 业务需求文档

## 项目概述

**项目名称**：Witsale (慧售)  
**项目类型**：多端企业销售管理系统（混合B2B+B2C模式）  
**核心用户**：企业员工、渠道客户、终端客户、最终客户  

---

## 一、用户角色与权限体系

### 1.1 用户角色定义

| 角色名称 | 说明 | 对应端 |
|---------|------|--------|
| 超级管理员 | 系统最高权限 | 内部管理端 |
| 企业管理员 | 企业内部管理 | 内部管理端 |
| 运营人员 | 商品、订单运营 | 内部管理端 |
| 财务人员 | 财务对账、结算 | 内部管理端 |
| 外勤人员 | 客户拜访、现场下单 | 外勤App |
| 渠道客户管理员 | 渠道客户内部管理 | 外部管理端 |
| 渠道客户员工 | 渠道客户业务操作 | 外部管理端 |
| 终端客户管理员 | 终端客户内部管理 | 外部管理端 |
| 终端客户员工 | 终端客户业务操作 | 外部管理端 |
| 最终客户 | 个人消费者 | 普通客户App + 商城Web |

## 九、需求梳理总结

### 已确认的核心信息

| 模块 | 内容 |
|------|------|
| 项目类型 | 多端企业销售管理系统（混合B2B+B2C） |
| 商品类型 | 全品类综合性销售平台 |
| 用户角色 | 企业员工 + 渠道客户 + 终端客户 + 最终客户 |
| 管理端 | 内部管理端 + 外部管理端（渠道/终端客户自用） |
| 价格体系 | 4级定价（企业/渠道/终端/最终客户）+ 特殊策略（达赠/礼品/运补） |
| 库存管理 | 分布式多级库存（企业仓 + 渠道仓 + 终端仓） |
| 支付方式 | 多种支付（微信/支付宝/银行转账/线下/账期等） |
| 财务结算 | 多种周期/方式支持，模块较复杂 |
| 外勤管理 | 打卡/轨迹/拜访等，具体细节开发时再细化 |
