# Witsale 后端实现计划

## 技术栈确认

- **Web框架**: FastAPI (Python)
- **数据库**: PostgreSQL 15+ (充分利用新特性)
- **缓存**: Redis 7+
- **架构模式**: DDD (领域驱动设计)
- **ORM**: SQLAlchemy 2.0 (异步)
- **数据迁移**: Alembic
- **认证**: JWT + OAuth2
- **API文档**: OpenAPI/Swagger (自动生成)
- **知识图谱/Ontology**: 
  - **owlready2**: OWL本体管理
  - **rdflib**: RDF数据处理
  - **可选**: Neo4j（图数据库，后续可考虑）

## 二、核心领域模块划分

### 2.1 用户与认证领域 (User & Auth)
- 用户实体 (企业员工、渠道商、终端客户、最终客户)
- 角色与权限管理 (RBAC)
- JWT认证
- OAuth2集成

### 2.2 客户领域 (Customer)
- 企业客户管理
- 渠道客户管理
- 终端客户管理
- 最终客户管理
- 客户分层与标签
- 客户关系维护

### 2.3 商品领域 (Product)
- 商品SKU管理
- 商品分类
- 商品库存
- 商品价格体系 (多级定价)
- 商品属性

### 2.4 订单领域 (Order)
- 订单创建与流转
- 订单状态机
- 多级订单 (渠道商/终端/最终客户)
- 订单支付
- 订单退款

### 2.5 库存领域 (Inventory)
- 仓库管理
- 库存盘点
- 库存调拨
- 库存预警

### 2.6 外勤领域 (Field Service)
- 外勤人员管理
- 客户拜访记录
- 现场订单录入
- 位置追踪

### 2.7 知识图谱/Ontology领域 (Knowledge)
- 商品本体建模（分类、属性、关联关系）
- 客户本体建模（分层、标签、行为模式）
- 语义推理引擎
- 知识图谱查询API
- 智能推荐（基于本体的商品推荐）
- 语义搜索增强

## 三、PostgreSQL 特性利用

### 3.1 数据类型
- **JSONB**: 存储灵活的扩展属性
- **Array**: 标签、图片列表等
- **Range Types**: 价格区间、时间区间
- **UUID**: 主键使用UUID

### 3.2 高级特性
- **CTE & Window Functions**: 复杂统计查询
- **Materialized Views**: 报表预计算
- **Triggers**: 数据一致性保证
- **Full Text Search**: 商品搜索
- **Partitioning**: 大表分区 (订单表按时间分区)

### 3.3 扩展
- **PostGIS**: 地理位置相关 (外勤位置)
- **pg_trgm**: 模糊搜索优化

## 四、Redis 应用场景

- **Session存储**: JWT黑名单
- **缓存**:

- **Rate Limiting**: API限流
- **消息队列**: 异步任务 (可选)

## 六、API设计原则

### 6.1 RESTful规范
- 资源命名: 复数形式 (如 `/api/v1/products`)
- HTTP方法正确使用: GET/POST/PUT/PATCH/DELETE
- 状态码规范: 200/201/400/401/403/404/500

### 6.2 版本管理
- URL版本: `/api/v1/...`
- 向后兼容策略

### 6.3 分页与排序
- 统一分页参数: `page`, `page_size`
- 排序参数: `order_by`, `order_dir`

### 6.4 错误响应
```json
{
  "error_code": "VALIDATION_ERROR",
  "message": "请求参数验证失败",
  "details": [...]
}
```

## 七、安全考虑

1. **认证**: JWT + 刷新令牌
2. **授权**: RBAC + 资源级权限
3. **输入验证**: Pydantic严格验证
4. **SQL注入防护**: ORM参数化查询
5. **XSS防护**: 输出转义
6. **Rate Limiting**: 接口限流
7. **敏感数据加密**: 密码bcrypt，其他敏感字段加密存储
8. **HTTPS**: 生产环境强制HTTPS
