# Witsale 慧售 - 规则使用指南

## 1. 规则概述

本指南旨在帮助您更好地理解和使用项目中的规则文件，特别是如何通过有效的提示词来确保AI助手按照预期执行任务。

### 核心规则文件

- **AI协作规则**：`docs/rules/ai-collaboration-rules.md` - 定义AI与开发者之间的协作流程
- **项目规则**：`docs/guides/development-process.md` - 定义项目的开发流程规范
- **AI助手规范**：`AGENTS.md` - 定义AI助手的行为规范和开发流程

## 2. 提示词最佳实践

### 2.1 基础提示词结构

**有效的提示词应包含以下元素**：

1. **明确的任务描述**：清楚说明您希望AI执行的具体任务
2. **规则引用**：明确要求AI遵循特定规则
3. **上下文信息**：提供必要的背景信息和约束条件
4. **期望输出**：描述您期望的输出格式和内容
5. **审批要求**：明确需要您批准后才能执行的操作

### 2.2 最佳提示词示例

#### 示例1：代码分析任务

```
请按照 `docs/rules/ai-collaboration-rules.md` 中的协作规则，对商品管理模块的代码进行全面分析。

具体要求：
1. 分析范围：`backend/app/application/services/product_service.py` 和相关文件
2. 重点关注：性能问题、安全漏洞、代码质量
3. 输出格式：结构化的分析报告，包含问题识别、优化机会、改进方案
4. 请在分析完成后提交报告，获得我的批准后再执行任何修改。
```

#### 示例2：功能开发任务

```
请按照 `docs/rules/ai-collaboration-rules.md` 中的协作规则，开发用户认证模块的密码重置功能。

具体要求：
1. 功能需求：实现密码重置邮件发送和验证功能
2. 技术要求：使用FastAPI、JWT、SMTP邮件服务
3. 输出格式：详细的实现方案，包含API设计、数据模型、代码结构
4. 请在方案设计完成后提交审批，获得我的批准后再进行编码实现。
```

#### 示例3：Bug修复任务

```
请按照 `docs/rules/ai-collaboration-rules.md` 中的协作规则，修复订单管理模块的库存扣减bug。

具体问题：
1. 订单创建时库存扣减不正确，导致库存负数
2. 涉及文件：`backend/app/application/services/order_service.py`

要求：
1. 分析问题原因并提出修复方案
2. 编写测试用例验证修复效果
3. 提交修复方案获得批准后再执行代码修改
```

### 2.3 提示词技巧

1. **具体明确**：避免模糊的描述，提供具体的文件路径、功能需求和技术要求
2. **结构化**：使用编号和列表使提示词清晰易读
3. **规则引用**：明确引用相关规则文件，确保AI遵循正确的流程
4. **审批要求**：明确指出需要批准的环节，确保AI不会擅自执行修改
5. **上下文完整**：提供足够的背景信息，帮助AI理解任务的上下文

## 3. 规则使用流程

### 3.1 启动任务

1. **明确任务目标**：清晰定义您希望完成的任务
2. **选择适用规则**：根据任务类型选择适用的规则文件
3. **构建提示词**：按照最佳实践构建有效的提示词
4. **发送指令**：向AI助手发送完整的提示词

### 3.2 执行流程

1. **代码分析**：AI对相关代码进行全面分析
2. **方案提交**：AI提交详细的分析报告和改进方案
3. **方案审批**：您审查并批准方案
4. **编码实现**：AI按照批准的方案执行编码
5. **验证测试**：您验证实现结果
6. **迭代优化**：AI分析更新后的代码，寻找新的优化机会

### 3.3 常见场景处理

#### 场景1：新功能开发

1. **提示词**：明确功能需求、技术要求和输出格式
2. **流程**：分析 → 方案设计 → 审批 → 编码 → 测试
3. **重点**：详细的技术方案和API设计

#### 场景2：Bug修复

1. **提示词**：描述具体问题、涉及文件和期望结果
2. **流程**：问题分析 → 修复方案 → 审批 → 修复 → 验证
3. **重点**：问题根因分析和测试用例

#### 场景3：性能优化

1. **提示词**：指定优化目标、性能指标和测试方法
2. **流程**：性能分析 → 优化方案 → 审批 → 实施 → 性能测试
3. **重点**：性能数据对比和优化效果验证

## 4. 规则执行验证

### 4.1 验证AI是否遵循规则

**检查要点**：

- AI是否生成了结构化的分析报告
- AI是否等待您的批准后再执行修改
- AI是否提供了详细的实施报告
- AI是否遵循了项目的代码风格规范

### 4.2 常见问题处理

#### 问题1：AI直接执行了代码修改

**解决方案**：

- 明确指出AI违反了协作规则
- 要求AI回滚修改
- 重新启动流程，强调审批要求

#### 问题2：分析报告不够详细

**解决方案**：

- 提供更具体的分析要求
- 明确指出需要分析的具体方面
- 要求AI补充缺失的分析内容

#### 问题3：方案不符合项目目标

**解决方案**：

- 明确指出方案的问题所在
- 提供具体的修改要求
- 要求AI重新提交改进后的方案

## 6. 最佳实践总结

### 6.1 提示词最佳实践

1. **明确规则引用**：每次任务都明确引用相关规则文件
2. **具体任务描述**：提供详细的任务描述和要求
3. **结构化格式**：使用编号和列表使提示词清晰易读
4. **审批要求明确**：明确指出需要批准的环节
5. **上下文完整**：提供足够的背景信息

### 6.2 流程执行最佳实践

1. **严格遵循流程**：按照规则定义的流程执行每个步骤
2. **文档记录完整**：确保所有分析、方案和实施过程都有文档记录
3. **质量验证充分**：对AI的实现结果进行充分的测试和验证
4. **持续迭代优化**：通过迭代循环不断改进代码和流程
5. **沟通及时有效**：及时反馈问题和改进建议

### 6.3 常见误区避免

1. **避免模糊指令**：不使用模糊的描述，确保AI理解具体要求
2. **避免跳过审批**：不允许AI在未获得批准的情况下执行代码修改
3. **避免忽略测试**：确保每个修改都有相应的测试验证
4. **避免文档缺失**：确保所有开发活动都有相应的文档记录
5. **避免流程跳过**：不跳过规则定义的任何流程步骤

## 7. 示例对话流程

### 示例：商品管理模块优化

**步骤1：启动任务**

```
请按照 `docs/rules/ai-collaboration-rules.md` 中的协作规则，对商品管理模块的代码进行全面分析，重点关注性能问题和代码质量。

分析范围：
- `backend/app/application/services/product_service.py`
- `backend/app/infrastructure/persistence/product_repository.py`

输出要求：
- 结构化的分析报告
- 详细的改进方案
- 明确的优先级建议
```

**步骤2：AI分析**

```
# 代码分析报告 - 商品管理模块

## 执行摘要
对商品管理模块的代码进行了全面分析，发现了2个主要性能问题和3个代码质量问题。

## 问题识别
1. **性能问题**：商品列表查询在大数据量时响应缓慢
2. **性能问题**：SKU价格查询重复数据库调用
3. **代码质量**：重复代码较多，可维护性差
4. **代码质量**：错误处理机制不完善
5. **代码质量**：缺乏单元测试覆盖

## 优化机会
1. **数据库优化**：添加适当的索引，优化查询语句
2. **缓存策略**：实现Redis缓存热点数据
3. **代码重构**：提取公共函数，减少重复代码
4. **错误处理**：增强异常处理机制
5. **测试覆盖**：增加单元测试覆盖率

## 改进方案
1. **方案A**：优化数据库查询（优先级：高）
   - 具体措施：添加索引，优化查询语句
   - 预期收益：响应时间减少50%
   - 风险：低

2. **方案B**：实现缓存机制（优先级：中）
   - 具体措施：使用Redis缓存热点数据
   - 预期收益：响应时间减少70%
   - 风险：中（需要考虑缓存一致性）

3. **方案C**：代码重构（优先级：低）
   - 具体措施：提取公共函数，优化代码结构
   - 预期收益：代码可维护性提高
   - 风险：低

## 优先级建议
1. 方案A（高）
2. 方案B（中）
3. 方案C（低）
```

**步骤3：方案审批**

```
我已审查分析报告，批准方案A（优化数据库查询）和方案B（实现缓存机制），请按照方案实施。

对于方案B，请特别注意缓存一致性问题，确保数据更新时缓存能够及时同步。
```

**步骤4：AI实施**

```
# 实施报告 - 商品管理模块优化

## 方案执行情况
已完成方案A（优化数据库查询）和方案B（实现缓存机制）的实施。

## 代码修改详情
- **文件1**：`backend/app/infrastructure/persistence/product_repository.py`
  - 添加了商品名称和分类ID的索引
  - 优化了商品列表查询语句，减少了不必要的关联查询
  - 实现了分页查询，避免一次性加载所有数据

- **文件2**：`backend/app/application/services/product_service.py`
  - 实现了Redis缓存机制
  - 添加了缓存键生成和管理逻辑
  - 实现了缓存一致性处理，在数据更新时自动失效缓存

## 测试结果
- **测试用例**：15个测试用例全部通过
- **响应时间**：从原来的2.5秒减少到0.6秒，提升了76%
- **内存使用**：减少了25%

## 性能改进
| 场景 | 改进前 | 改进后 | 提升比例 |
|------|--------|--------|----------|
| 商品列表（1000条） | 2.5s | 0.6s | 76% |
| 商品搜索 | 1.8s | 0.4s | 78% |
| 分类筛选 | 1.2s | 0.3s | 75% |

## 潜在风险
- **风险**：缓存机制可能增加系统复杂度
- **应对措施**：添加了详细的缓存管理文档和监控机制

## 后续建议
1. 考虑实施方案C（代码重构）
2. 增加更多的性能测试用例
3. 优化商品图片的存储和访问
```

**步骤5：验证与迭代**

